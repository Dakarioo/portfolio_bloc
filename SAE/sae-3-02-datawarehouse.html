SAE 3.02

Intégration de données dans un datawarehouse

Reverse engineering de la base Chinook, modélisation MCD/MLD/MPD, création d'une base Oracle, conception d'un schéma en étoile, alimentation du datawarehouse via scripts SQL et rédaction d'un cahier de recette.

Fiche SAE

  • Titre : SAE 3.02 – Intégration de données dans un datawarehouse
  • Conditions : travail en binôme
  • Rendus : rapport, soutenance, repository GitHub (tous livrables)
  • Organisation : mise en place et suivi d'un planning (tâches, estimation, responsable, échéance) mis à jour tout au long du projet
  • Pré-requis : SQL (requêtes), bases de la modélisation (MCD/MLD/MPD), compréhension d'un système d'intégration et d'un schéma relationnel

Contexte / Problème

L'objectif est de partir d'une base de production existante (Chinook) afin de : comprendre son organisation, reconstruire sa modélisation, l'implémenter en Oracle, puis construire un entrepôt (schéma en étoile) pour faciliter l'analyse décisionnelle et la restitution.

Données & Sources

Chinook Database : base qui simule la vente de musique (artistes, albums, titres, factures, clients). Elle couvre notamment les entités : artist, album, track, invoice, invoiceline, customer.

Source : Chinook Database (GitHub)

Objectifs

  • Conception : comprendre la base, produire dictionnaire de données + MCD/MLD/MPD
  • Implémentation Oracle : scripts de création et d'alimentation de la base
  • Analyse SQL : requêtes répondant à une liste de questions métier
  • Datawarehouse : concevoir un modèle en étoile et créer l'entrepôt
  • Alimentation : scripts d'alimentation + cahier de recette (tests, contrôles)
  • Industrialisation : dépôt GitHub complet, structuré, documenté

Méthodologie

  1. 1
    Reverse engineering (conception)

    Analyse du schéma relationnel existant et des dépendances entre tables. Production des livrables : dictionnaire de données, MCD, MLD, MPD.

  2. 2
    Création & alimentation de la base Oracle

    Écriture des scripts SQL : tables, clés primaires/étrangères, contraintes. Écriture des scripts d'alimentation pour injecter les données dans la base Oracle.

  3. 3
    Analyse SQL (questions métier)

    Mise en place de requêtes SQL pour répondre à des questions d'analyse : pays avec le plus grand nombre de clients, client ayant dépensé le plus, genre le plus représenté, pourcentage de chansons jamais achetées, classement des pistes par durée au sein de chaque album.

  4. 4
    Création & alimentation du Datawarehouse

    Conception d'un schéma en étoile (faits + dimensions) adapté aux analyses. Scripts SQL de création de l'entrepôt, scripts d'alimentation, puis rédaction d'un cahier de recette : règles, contrôles, cas de test, résultats attendus.

Livrables

  • Dictionnaire de données (base de production)
  • MCD / MLD / MPD de la base
  • Scripts SQL : création Oracle + alimentation Oracle
  • Requêtes d'analyse (liste de questions)
  • Modèle en étoile + scripts de création de l'entrepôt
  • Scripts d'alimentation du datawarehouse
  • Cahier de recette (tests + contrôles)
  • Rapport + supports de soutenance

BCC Mobilisés

BCC TRAITER

  • Nettoyage/normalisation lors de l'intégration
  • Contrôles d'intégrité (clés, doublons, cohérence des volumes)
  • Alimentation de la base Oracle et de l'entrepôt

BCC ANALYSER

  • Questionner la donnée via SQL (indicateurs, classements, agrégations)
  • Interpréter des résultats (top clients, genres dominants, titres non vendus)
  • Identifier les questions métier à adresser via le datawarehouse

BCC VALORISER

  • Documentation (dictionnaire de données, modèle, cahier de recette)
  • Structuration du dépôt GitHub
  • Présentation et soutenance du travail

BCC MODÉLISER / DÉVELOPPER

  • Rétro-conception (MCD/MLD/MPD)
  • Conception DW : faits / dimensions, clés de substitution
  • Écriture et structuration des scripts SQL

Compétences transversales

  • Gestion de projet : planning, répartition des tâches, suivi d'avancement
  • Travail en binôme : coordination, revue croisée, résolution de problèmes
  • Rigueur et traçabilité : dépôt GitHub, documentation, tests

Bilan

  • Maîtrise des scripts SQL (création, intégration, analyse)
  • Vision complète "production → entrepôt → exploitation"
