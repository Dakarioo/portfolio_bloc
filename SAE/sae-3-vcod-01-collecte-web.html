SAE_VCOD

MonVoyage — Application Python de planification de séjour

Application Python complète qui automatise la collecte d'informations touristiques (hôtels, activités, météo, transports) via API et web scraping, génère des exports CSV et une carte interactive Folium.

Contexte / Problématique

Planifier un séjour nécessite de centraliser des informations fiables provenant de sources variées (hôtels, restaurants, lieux d'intérêt, météo, transport). Le problème principal est de rendre ces données exploitables : automatiser la collecte, gérer les formats hétérogènes, filtrer les résultats pertinents et produire une restitution claire.

Objectifs

  • Automatiser la collecte d'informations via API et scraping.
  • Uniformiser et structurer des données hétérogènes (champs, types, formats).
  • Exporter les résultats en CSV (réutilisables pour analyse/dashboards).
  • Générer une carte interactive des points d'intérêt (Folium).
  • Assurer la robustesse : gestion d'erreurs réseau, absence de résultats, champs manquants.

Sources de données & APIs

  • Hôtels (OSM / Opendatasoft) : public.opendatasoft.com/explore/assets/osm-france-tourism-accommodation/
  • Activités / lieux (OSM / Opendatasoft) : public.opendatasoft.com/explore/assets/osm-france-historic/
  • Restauration (OSM / Opendatasoft) : public.opendatasoft.com/explore/assets/osm-france-food-service/
  • Transport / distance / itinéraires : openrouteservice.org/
  • SNCF (données / infos) : ressources.data.sncf.com/explore/dataset/api-sncf/information/
  • Météo : openweathermap.org/api
  • Description de la ville (scraping) : fr.wikipedia.org/wiki/Lille

Les sources sont combinées dans un même flux pour produire une sortie structurée centrée sur Lille.

Pipeline de traitement

  1. 1
    Collecte — Requêtes HTTP (requests) + parsing HTML (BeautifulSoup) sur toutes les sources.
  2. 2
    Normalisation — Nettoyage des champs, formats cohérents (coordonnées, noms, catégories).
  3. 3
    Structuration — Objets/structures JSON internes + organisation par catégories (hôtels / restos / lieux).
  4. 4
    Exports CSV — Fichiers CSV par ville et par catégorie (ex. hotels_lille.csv), réutilisables pour analyses et dashboards.
  5. 5
    Restitution carte Folium — Carte interactive avec marqueurs par catégorie (et regroupements si besoin) + gestion des erreurs (timeouts, réponses incomplètes, 0 résultat).

BCC mobilisés

BCC ANALYSER

Définition de critères de pertinence (ville, catégories), contrôle de cohérence des résultats, identification des limites (couverture/qualité).

BCC TRAITER

Nettoyage et normalisation des données, gestion des valeurs manquantes, structuration JSON → CSV, fiabilisation des formats.

BCC VALORISER

Restitution claire : exports CSV réutilisables + carte Folium, synthèse et lisibilité pour un utilisateur non technique.

BCC MODÉLISER / DÉVELOPPER

Conception d'une application structurée (POO), séparation des responsabilités, gestion des erreurs et reproductibilité.

Compétences métier mises en œuvre

  • Collecte automatisée via API (requests) et traitement de réponses JSON.
  • Web scraping (BeautifulSoup) : extraction et structuration d'informations de pages web.
  • Organisation et structuration des données : schéma de sortie cohérent (catégories / champs), répartition du travail.
  • Exports CSV (réutilisables pour analyse et dashboards).
  • Cartographie (Folium) : restitution visuelle des points d'intérêt.
  • Fiabilisation : gestion d'erreurs, cas limites, résultats vides.

Compétences transversales

  • Rigueur : conventions de nommage, formats homogènes, contrôle des sorties.
  • Organisation : séparation des modules (collecte / traitement / export / carte).
  • Communication : documentation, explication du pipeline et des choix techniques.

Bilan personnel

Acquis / points forts

  • Construction d'un flux complet : collecte → traitement → export → restitution.
  • Meilleure maîtrise des cas réels (réponses incomplètes, erreurs réseau, changements de structure).
  • Sorties réutilisables (CSV) et présentation claire (carte).

Difficultés / points à retravailler

  • Stabilité du scraping (si la page change) : prévoir des sélecteurs robustes.
  • Standardiser certaines informations selon les sources (noms, catégories, champs manquants).
  • Améliorer l'automatisation et la maintenabilité du code.
