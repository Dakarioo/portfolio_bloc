SAE_VCOD

MonVoyage - Application Python de planification (Séjour à Lille)

Conception d’une application Python qui automatise la collecte d’informations (hôtels, activités, restauration, météo, transports) et génère une sortie exploitable : exports CSV + carte interactive (Folium), avec une logique de traitement fiable (gestion d’erreurs, normalisation, structuration des résultats).

Python requests BeautifulSoup folium csv json API Web scraping

Contexte / Problématique

Planifier un séjour nécessite de centraliser des informations fiables provenant de sources variées (hôtels, restaurants, lieux d’intérêt, météo, transport). Le problème principal est de rendre ces données exploitables : automatiser la collecte, gérer les formats hétérogènes, filtrer les résultats pertinents et produire une restitution claire.

Objectifs

  • Automatiser la collecte d’informations via API et scraping.
  • Uniformiser et structurer des données hétérogènes (champs, types, formats).
  • Exporter les résultats en CSV (réutilisables pour analyse/dashboards).
  • Générer une carte interactive des points d’intérêt (Folium).
  • Assurer la robustesse : gestion d’erreurs réseau, absence de résultats, champs manquants.

Sources de données & APIs utilisées

  • Hôtels (OSM / Opendatasoft) : https://public.opendatasoft.com/explore/assets/osm-france-tourism-accommodation/
  • Activités / lieux (OSM / Opendatasoft) : https://public.opendatasoft.com/explore/assets/osm-france-historic/
  • Restauration (OSM / Opendatasoft) : https://public.opendatasoft.com/explore/assets/osm-france-food-service/
  • Transport / distance / itinéraires : https://openrouteservice.org/
  • SNCF (données / infos) : https://ressources.data.sncf.com/explore/dataset/api-sncf/information/
  • Météo : https://openweathermap.org/api
  • Description de la ville (scraping) : https://fr.wikipedia.org/wiki/Lille

Les sources sont combinées dans un même flux pour produire une sortie structurée centrée sur Lille.

Pipeline de traitement

  • Collecte : requêtes HTTP (requests) + parsing HTML (BeautifulSoup).
  • Normalisation : nettoyage des champs, formats cohérents (ex. coordonnées, noms, catégories).
  • Structuration : objets/structures JSON internes + organisation par catégories (hôtels/restos/lieux).
  • Exports : fichiers CSV par ville et par catégorie (ex. hotels_lille.csv).
  • Restitution : carte Folium avec marqueurs (et regroupements si besoin).
  • Robustesse : gestion des erreurs (timeouts, réponses incomplètes, 0 résultat).

BCC VALORISER

BCC MODÉLISER / DÉVELOPPER

Compétences métier mises en œuvre

  • Collecte automatisée via API (requests) et traitement de réponses JSON
  • Web scraping (BeautifulSoup) : extraction et structuration d’informations de pages web
  • Organisation et structuration des données : schéma de sortie cohérent (catégories / champs), répartition du travail
  • Exports CSV (réutilisables pour analyse et dashboards)
  • Cartographie (Folium) : restitution visuelle des points d’intérêt
  • Fiabilisation : gestion d’erreurs, cas limites, résultats vides

Bilan personnel

Acquis / points forts

  • Meilleure maîtrise des cas réels (réponses incomplètes, erreurs réseau, changements de structure).

  • Améliorer l’automatisation.
